# design-sync notes — Sweeper

Repo-specific gotchas for future syncs of this design system.

## Shape & build

- **This is a Next.js app, not a published component package.** There is no `dist/` and no library build script — the converter runs in **synth-entry mode** from `components/` (`srcDir: "components"`).
- All `@/lib/...` imports inside `components/*.tsx` are **`import type`** (erased at compile), so the component bundle is self-contained React + the four component files. No `@solana/web3.js` / engine code is pulled into the bundle.
- Path alias `@/* -> ./*` is resolved via `tsconfig.json` (`cfg.tsconfig`). The components import each other as `@/components/charts` etc.
- `"use client"` directives at the top of panels/replay are harmless to esbuild.

## CSS / design tokens

- The design language is **Tailwind v4 `@theme` tokens + custom component classes** defined in `app/globals.css` (`.panel`, `.chip`, `.btn`/`.btn-primary`/`.btn-danger`, `.field`, `.eyebrow`, `.tnum`, `.row-hover`, `.pulse-dot`, `.flash-up/down`). Color tokens: `--color-brand` (#c8f751 lime), `--color-cyan`, `--color-up`, `--color-down`, `--color-warn`, `--color-crit`, `--color-info`, `--color-ink/muted/faint`, `--color-bg/panel/panel2/line/line2`.
- `cfg.cssEntry` points at `.design-sync/_compiled.css`, produced by `cfg.buildCmd` (Tailwind v4 CLI compiling `app/globals.css` with source detection over the repo). The CLI is installed into the isolated `.ds-sync/node_modules` (not the repo lockfile).
- Dark theme: `--color-bg` is near-black (#0a0b0f). Previews must render on a dark background or text/tokens read as invisible — the bundle's body styling applies `background: var(--color-bg)`.

## Known render warns (triaged — not new on re-sync)

- **[RENDER_THIN] EquityChart, Sparkline** — both are pure-SVG charts with no text nodes, so the render-check heuristic reports "no text / paints nothing". They DO paint (verified in the review sheets: EquityChart draws 4 colored polylines + baseline; Sparkline draws its curve). Benign.
- **[GRID_OVERFLOW] → cardMode column** is applied via `cfg.overrides` for the wide components: Arena, AuditTrail, EquityChart, OddsBoard, ReplayLab, ScoreHeader, SentinelFeed. Expected; the column cards can't re-flag `wide`.

## Preview techniques (so a re-sync understands the authored previews)

- **ProofModal** fetches `/api/proof/<seq>`; the preview mocks `window.fetch` to return a canned **verified** `ProofBundle` so the success state renders. It's also a `position: fixed` full-viewport overlay — the preview wraps it in a `transform: translateZ(0)` box (a containing block for the fixed overlay) so it centers inside the card. `cfg.overrides.ProofModal` = `{cardMode: single, viewport: 700x600}`.
- **SentinelFeed / OddsBoard / AuditTrail** are wrapped in a fixed-width `<div>` in their previews because they use `flex h-full` / wide tables and need a width to lay out.
- Domain fixtures (agents, ticks, signals, ledgers, settlements) are hand-built in `.design-sync/previews/*.tsx` from the real types — World Cup-flavored (Argentina/France, etc.).

## Re-sync setup (REQUIRED — the build won't run without this)

- The converter resolves the package dir as `node_modules/<pkg>` = `node_modules/sweeper`, which doesn't exist because this repo *is* the package. Create a **self-symlink** before building:
  `ln -sfn "$(pwd)" node_modules/sweeper`
  (gitignored, recreated per clone — like the converter deps). Without it the build dies with `ENOENT … node_modules/sweeper/package.json`. Passing `--entry` instead is NOT a fix here: it disables synth-entry src discovery and yields zero components.
- Build command sequence: `ln -sfn "$(pwd)" node_modules/sweeper` → run `cfg.buildCmd` (compiles `_compiled.css`) → `node .ds-sync/package-build.mjs --config .design-sync/config.json --node-modules ./node_modules --out ./ds-bundle` (NO `--entry`).
- Playwright/Chromium for the render check live in the macOS default cache (`~/Library/Caches/ms-playwright`). Do NOT set `PLAYWRIGHT_BROWSERS_PATH=0` — that makes validate look in `.ds-sync/node_modules` where the browser isn't.

## Re-sync risks

- `_compiled.css` is regenerated by `buildCmd`; if a future Tailwind version changes source-detection defaults, verify the compiled CSS still contains the utilities used by the components (grep for `.panel`, `text-brand`, `items-center`). The original reference was the Next build output at `.next/static/css/*.css` (~17KB).
- Components are domain-coupled (props are `EngineState`, `TickView`, `Signal[]`, `AgentView[]`, `SettlementReceipt`, `ProofBundle`). Preview fixtures live in `.design-sync/previews/*.tsx` and were hand-built from the type shapes in `lib/engine/state.ts`, `lib/sentinel/types.ts`, `lib/execution/portfolio.ts`, `lib/proof/`. If those types change, the preview props may need updating.
- `ProofModal` and `ReplayLab` fetch from `/api/*`; in static previews those fetches fail and the component shows its loading/empty state. Authored previews render the empty/initial state deliberately.
